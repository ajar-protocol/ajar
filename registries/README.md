# /registries

Machine-readable protocol registries — populated in Phase 0:
- `scopes.md` — mandate scope registry v1 (T0.3): `content.*`, `commerce.*`, `communication.*`, `account.*`, `data.*`
- `error-codes.md` — `Ajar-Error-Code` registry (T0.6), RFC 9457 profile
- `settlement-adapters.md` — registered settlement adapter identifiers (`x402`, `ap2-card`, ...)

Registry changes follow the AEP process once v0.1 ships. Additions must include semantics, examples, and (for scopes) matching-rule notes.
